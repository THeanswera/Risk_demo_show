# Калькулятор ИПР по Приказу МЧС №1140

Streamlit-приложение для расчёта индивидуального пожарного риска (ИПР)
по Методике определения расчётных величин пожарного риска
(Приказ МЧС России от 14.11.2022 № 1140).

## Структура проекта

```
app.py              - точка входа Streamlit
core/
  constants.py      - константы (R_NORM, K_STD, K_AP_VALUE и т.д.)
  formulas.py       - формулы (1)–(7) Методики
  compute.py        - основная логика расчёта ИПР
  tables.py         - справочные таблицы (Прил. 3, 4)
  t_block.py        - расчёт tбл (Прил. 1, раздел IV)
  evacuation.py     - расчёт tр и tск (Прил. 6, параметры по Прил. 2)
ui/
  scenarios.py      - шаг 1: сценарии пожара
  groups.py         - шаг 2: группы эвакуируемого контингента
  results.py        - шаг 3: результаты расчёта
  doors.py          - заглушка (двери НЕ реализованы)
  sidebar.py        - боковая панель (справочники, слайдеры, расчёт tбл, расчёт tр)
  evacuation.py     - UI расчёта tр по Приложению 6
  reference.py      - справочные таблицы и формулы (expander)
export/
  report_docx.py    - генерация Word-отчёта
visualization/
  gauge.py          - HTML-компонент шкалы риска
utils/
  helpers.py        - вспомогательные функции
tests/
  test_formulas.py  - unit-тесты формул
  test_compute.py   - интеграционные тесты compute_all()
  test_evacuation.py - тесты расчёта tр и tск (Прил. 6)
```

## Критические правила (не нарушать!)

- **K_обн, K_СОУЭ, K_ПДЗ** — строго 0 или 0.8 (п. 41, 44, 45). Реализованы через CheckboxColumn.
- **K_ап** — строго 0 или 0.9 (п. 15). Реализован через CheckboxColumn.
- **Агрегирование** — через max, не sum (формулы 2, 3).
- **P_э <= 0.999** — всегда (формула 6, п. 17).
- **Единицы**: tнэ по П4.1 считается в секундах, потом делится на 60 для формулы (6).
- **Приложение 1** -> tбл, **Приложение 6** -> tр. Не путать!
- **tр** рассчитывается по Приложению 6 (формула П6.1), параметры движения по Приложению 2.
- **tск** определяется автоматически при обнаружении скоплений (qi > qmax).
- **Противопожарные двери** — НЕ реализованы (формула 8). Пользователь моделирует через отдельные сценарии с разными tбл.
